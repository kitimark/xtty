# `xtty` Phased Build Plan

> **Provenance:** Drafted 2026-06-27 from the [stack sketch](01-stack-sketch.md), [requirements](../03-analysis/xtty-requirements.md), and [internals](../02-internals/) research. **Re-mapped 2026-06-27** to the **SwiftTerm adoption decision: staged, start at Level 3** (see the stack sketch's Spike findings ‚Üí Decision). Earlier drafts assumed a build-it-ourselves renderer; that work is now deferred to a *conditional* late phase.

## Principle: reuse the engine + view, build the differentiators

Starting at **Level 3** (wrap SwiftTerm's `TerminalView`), the PTY loop, VT parsing, rendering, input, selection, scrollback, search, and graphics protocols are **already done**. So the early milestones collapse into "integrate & configure," and real effort moves up to what makes xtty *xtty*: tabs/splits, OSC 133 blocks, the session sidebar, and the file/diff view.

**The load-bearing rule (set at P0):** all xtty logic talks to the **`Terminal` engine** (`view.getTerminal()`), never to `TerminalView` internals ‚Äî so the render layer stays swappable and the L3‚ÜíL1 escape hatch (P8) is a contained refactor, not a rewrite.

Requirement tags reference [xtty-requirements](../03-analysis/xtty-requirements.md) (M = must-have, H = high-value, N = nice-to-have).

## Phase 7 ‚Äî Polish + MEASURE (decision gate)  ¬∑  M1, M4 *(old P10)*  *(üî¨ P7a `add-latency-memory-harness` ‚úÖ archived; P7b `add-trustworthy-latency-probe` ‚úÖ implemented, benched & archived 2026-06-29 ‚Üí **renderer decided: keep CoreGraphics, skip Phase 8**; P7c implemented & archived 2026-06-29 ‚Äî `add-lifecycle-census`)*
**Goal:** verify the lean + fast requirements with data ‚Äî this gates Phase 8.
- **Measure** key-to-photon latency and memory (scrollback + atlas + panes) against M1/M4.
- If short: first flip `useMetalRenderer` + tune frame pacing (cheap); re-measure.
- Memory pass: scrollback cap, retain-cycle/leak audit (Instruments). Crash hardening.
- **Hardened Runtime + Developer ID + notarization** for distribution.

**Explore-phase decisions (2026-06-29 ‚Äî [`p7-measurement-methodology`](../03-analysis/p7-measurement-methodology.md)):** the bar is undefined (M1/M4 qualitative) ‚Üí set it **relative** to installed comparators (Warp/iTerm2/Terminal.app; no Ghostty). Latency via a **fork-free in-process screen-capture probe** (inject `CGEvent` ‚Üí poll window pixels until the glyph changes; renderer-agnostic, the CoreGraphics-vs-Metal **delta is exact** despite excluding the ~20 ms hardware tail) rather than a per-renderer SwiftTerm patch. Memory via `task_info` under fixed scenarios; the 0.15 s state-dump timer **reports** aggregates but latency **capture** needs its own high-res path. A base-only `renderer = coregraphics|metal` config key (+ `-UITestRenderer` arg) gives rebuild-free A/B. **Distribution is deferred** ‚Äî 0 codesigning identities on the dev machine, and it's orthogonal to the gate. **Recommended split:** **P7a `add-latency-memory-harness`** (gate-critical, fork-free, a reusable regression guard) ‚Üí **P7b** renderer decision (A/B + comparators ‚Üí decision doc) ‚Üí **P7c** memory/leak pass; distribution as a separate later change. Scope the first proposal to **P7a** and let its numbers drive P7b.

**P7a shipped (implemented & archived 2026-06-29 ‚Äî see the [apply addendum](../03-analysis/p7-measurement-methodology.md)):** the fork-free harness ‚Äî `renderer = coregraphics|metal` key + SwiftTerm `setUseMetal` A/B, a `task_info` memory sampler over independently-reset scenarios, an in-process latency probe (ScreenCaptureKit one-shot; `CGWindowListCreateImage` turned out unavailable on the SDK), a `-Benchmark` mode + `make bench` writing a JSON report, the new `performance-harness` spec, 224 unit + 31 UI tests. **Findings:** ‚úÖ **memory is lean & trustworthy** ‚Äî idle ~68 MB footprint, 0 % idle CPU, ‚â§135 MB saturated (M1 looks met); ‚ùå the **screenshot-polling latency probe is too coarse** (capture ~20 ms > the ~8‚Äì16 ms signal ‚Üí can't resolve key-to-photon, can't distinguish CoreGraphics from Metal), so the **trustworthy latency probe + the renderer verdict move to P7b**. A companion **`add-local-signing-identity`** change added an opt-in stable self-signed signing identity so the latency probe's Screen-Recording TCC grant persists across rebuilds (the default stays ad-hoc). **P7b/P7c remain;** distribution (Hardened Runtime + Developer ID + notarization) still deferred.

**P7b methodology (2026-06-29 ‚Äî explore `p7b`; [P7b addendum](../03-analysis/p7-measurement-methodology.md#addendum-2026-06-29--p7b-the-trustworthy-probe-methodology-the-renderer-is-a-wash-the-throttle-is-the-lever); ‚Üí implemented & archived, verdict below):** a 14-agent research workflow + a SwiftTerm-checkout deep-read resolved all 7 open questions. The trustworthy probe = a continuous **`SCStream` reading per-frame `displayTime`** (on-glass, fork-free, ~90 % P7a reuse; the next `.complete` frame after t0 *is* the keystroke), with a load-bearing **clock-unit fix** (`displayTime` is mach *ticks* not ns) + a **startup epoch-calibration gate**. ‚öÝÔ∏è **Headline reframe:** both renderers sit behind a **shared ~16.67 ms output-coalescing throttle** (`queuePendingDisplay`) ‚Üí the A/B is near a **wash at the median**, and that throttle (not the renderer) is xtty's latency lever. Comparators = Terminal.app-first stretch (iTerm2 Secure-Event-Input blocks injection); no hardware rig needed (omitted tail cancels in the delta).

**P7b decided (2026-06-29 ‚Äî `add-trustworthy-latency-probe`, implemented & benched 3√ó/renderer; [apply-result addendum](../03-analysis/p7-measurement-methodology.md#addendum-2026-06-29--p7b-apply-result-the-trustworthy-probe-works-verdict--keep-coregraphics-skip-phase-8)):** the `SCStream`-`displayTime` probe (clock-fixed, calibration-gated ‚Äî passed, offset <4 ms) resolved the renderers P7a couldn't. **Verdict: keep CoreGraphics, skip Phase 8** ‚Äî CG measured faster median (~31 vs ~33 ms), much tighter tail (p99 ~40 vs ~50‚Äì120 ms), and leaner memory (~55 vs ~62 MB idle) than SwiftTerm's experimental Metal path. The shared ~16.7 ms `queuePendingDisplay` throttle dominates the median (the real future latency lever); the probe also exposed Metal's worse tail. **Phase 8 is therefore NOT needed.** Remaining: **P7c** (Instruments leak/retain pass), distribution deferred.

**P7c implemented & archived (2026-06-29 ‚Äî `add-lifecycle-census`, new `lifecycle-census` spec; researched via `/opsx:explore p7c` + a 14-agent workflow ‚Üí [`p7c-leak-retain-audit`](../03-analysis/p7c-leak-retain-audit.md)):** the leak/retain pass. **Headline: no leak to hunt** ‚Äî both explore-flagged retain cycles (`WindowController‚áÑGitReviewController`, `PaneController‚áÑLinkRoutingTerminalDelegate`) **refuted** (all closures `[weak self]`, delegates `weak`, `GitReviewTarget` a transient value type), App-layer sweep clean bar 2 cosmetic nits. Reframed to **prove + guard + profile-SwiftTerm**, shipped as: a DEBUG **live-instance census** (`nonisolated(unsafe) static liveCount` on 6 lifecycle types ‚Üí a `liveInstanceCounts` state-dump field) + a **churn XCUITest** asserting return-to-baseline (the gated net ‚Äî **proven** by injecting a deliberate retain ‚Üí failed `baseline 1, final 8`, then reverted) + in-process **weak-sentinel** dealloc tests for `XttyCore` types + a diagnostic-not-gate **`make audit-leaks`** (`leaks` reports **0 leaks at idle**). The two nits fixed (`dumpTimer.invalidate()`, `BenchmarkRunner [weak controller]`). SwiftTerm's **unbounded glyph/font caches** stay a documented third-party watch-item (fork-free; not patched). Footprint-after-churn stays **report-only** (too noisy); **dropped** the proposed `max-memory-mb` config key.
